# TMP102 Temperature Sensor — Dual NUCLEO-F446RE Communication

Real-time temperature acquisition and transmission between two STM32 NUCLEO-F446RE boards, developed as part of the **EE3023 Embedded Systems** module (B.E. Electrical & Electronic Engineering).

## System Overview

A master board reads temperature data from a TMP102 sensor and forwards it to a slave board, which decodes and outputs the result to a serial monitor.

```
TMP102 ──I2C+DMA──► Master NUCLEO-F446RE ──SPI──► Slave NUCLEO-F446RE ──UART──► PC (PuTTY)
```

The master runs two concurrent FreeRTOS tasks coordinated via a queue:
- **I2C task** — triggers a DMA read from the TMP102, packs the 12-bit result into 2 bytes, and enqueues it
- **SPI task** — dequeues the data and transmits it to the slave board

The slave receives the 2 bytes over SPI, reconstructs the 12-bit temperature value, converts it to Celsius, and sends it over UART at 115200 baud.

## Key Technical Areas

| Area | Detail |
|---|---|
| MCU | STM32F446RE (ARM Cortex-M4, 180 MHz) |
| RTOS | FreeRTOS — task scheduling, inter-task queue |
| Sensor interface | I2C with DMA (non-blocking, interrupt-driven via semaphore) |
| Board-to-board | SPI full-duplex, GPIO-controlled slave select |
| Host output | UART at 115200 baud |
| IDE | STM32CubeIDE, HAL library |

## Repository Structure

```
├── Master/
│   ├── main.c      # FreeRTOS task definitions (I2C + SPI tasks, queue setup)
│   ├── temp.c      # I2C+DMA communication, TMP102 decoding, UART output
│   └── temp.h      # TMP102 register definitions, function prototypes
└── Slave/
    ├── main.c      # SPI receive loop
    ├── temp.c      # Temperature reconstruction from SPI bytes, UART output
    └── temp.h      # Definitions and prototypes
```

> **Note:** `main.c` files contain only the user-written sections. The HAL peripheral initialisations (clock, GPIO, DMA, I2C, SPI, UART) are auto-generated by STM32CubeIDE from the `.ioc` configuration file and are not included here.

## Notable Implementation Details

- **DMA + semaphore:** `HAL_I2C_Mem_Read_DMA` initiates a non-blocking read; the I2C task suspends via `xSemaphoreTake` and the CPU is free until the DMA completion interrupt fires `xSemaphoreGiveFromISR`, avoiding a busy-wait.
- **FreeRTOS queue:** the two tasks are decoupled through a 10-slot binary queue (`uint8_t[2]` items), so the I2C task can acquire the next reading while the SPI task is still transmitting.
- **12-bit packing:** the TMP102 returns a 12-bit value; the master splits it across 2 bytes for SPI (`byte[0] = val >> 4`, `byte[1] = (val & 0xF) << 4`) and the slave reassembles it before conversion.
- **Negative temperature:** 2's complement sign extension is applied before the Celsius conversion, with separate handling of the sign character for correct UART output below 0 °C.
